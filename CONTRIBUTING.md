# Development Workflow

## Branch Naming
- Feature branches: `feature/<github-username>/<issue-description>`
- Fix branches: `fix/<github-username>/<issue-description>`
- Branch names must be written in English.

## Commit Style
- Use Angular commit style (feat, fix, docs, chore, test, refactor) in imperative form.
- Commit messages must be written in English.

## Pull Request Process
- Create a branch from main, make changes, open a PR, request review, resolve comments, and merge after approval.

## Issues
- Assign an owner, link PRs, and use labels.

## Code Quality
- Requirement before opening a PR: `docker compose build`
